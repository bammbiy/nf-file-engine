# NF File Engine

신문 PDF와 반복 문서 파일명을 빠르게 정리하기 위한 데스크톱 파일 관리 도구입니다.  
드래그하듯 파일을 추가하고, 날짜/매체코드/페이지 규칙으로 변경 예정 파일명을 확인한 뒤 한 번에 rename 할 수 있습니다.

## 왜 만들었나

회사 업무에서 반복되는 파일명 변경 작업은 단순하지만 실수가 나기 쉽습니다.  
NF File Engine은 이 과정을 `미리보기 -> 실행 -> 히스토리 기록 -> 되돌리기` 흐름으로 관리해서, 빠르게 처리하면서도 실수했을 때 복구할 수 있게 만드는 것을 목표로 합니다.

이 프로젝트는 특정 회사 프로그램을 복제하는 것이 아니라, PDF/계약서/보고서/스캔 문서 등 여러 문서 파일에 적용할 수 있는 범용 문서 파일명 관리 엔진으로 확장하기 위해 시작했습니다.

## 주요 기능

- PDF 및 일반 파일 다중 선택
- `{DATE}_{MEDIA}_{PAGE}` 형식의 파일명 규칙 생성
- 날짜, 매체코드, 페이지 번호 자동 정규화
- 변경 전 대상 파일명 미리보기
- 파일명 충돌 및 중복 대상 감지
- SQLite 기반 rename 히스토리 저장
- 마지막 변경 작업 되돌리기
- 외부 패키지 없이 실행 가능한 기본 GUI

## 파일명 규칙

기본 규칙은 다음과 같습니다.

```text
{DATE}_{MEDIA}_{PAGE}
```

예시:

```text
20260628_ja00_001.pdf
```

규칙 입력창에서 아래처럼 바꿀 수도 있습니다.

```text
{MEDIA}-{DATE}-{PAGE}
```

결과:

```text
ja00-20260628-001.pdf
```

## 실행 방법

Python 3.11 이상을 권장합니다.

```bash
cd nf-file-engine
python src/app.py
```

현재 MVP는 Python 표준 라이브러리만 사용합니다. 별도의 패키지 설치가 필요 없습니다.

## 개발 로드맵

### 1단계: NF File Engine MVP

- 파일 선택
- 파일명 규칙 적용
- 변경 전 미리보기
- 일괄 rename
- 히스토리 저장
- 마지막 변경 취소

### 2단계: Folder Watcher

- 다운로드 폴더 감시
- 새 PDF 자동 감지
- rename 후보 자동 생성
- 처리 완료 알림

### 3단계: OCR Inspector

- PDF 텍스트 존재 여부 검사
- OCR 필요 파일 분류
- OCR 처리 큐 연동

### 4단계: Upload Monitor

- FTP 업로드 상태 확인
- 실패/성공/재시도 이력 관리
- 작업 현황 대시보드 제공

### 5단계: NewsFlow

NF File Engine, Folder Watcher, OCR Inspector, Upload Monitor를 하나의 문서 처리 파이프라인으로 통합합니다.

## 포트폴리오 포인트

이 프로젝트는 단순 파일명 변경 도구에서 시작하지만, 장기적으로는 콘텐츠 제작 업무에서 발생하는 문서 처리 흐름을 자동화하는 데이터 파이프라인으로 확장할 수 있습니다.

핵심 스토리는 다음과 같습니다.

```text
반복 파일 작업 자동화
-> 변경 이력과 복구 기능 추가
-> 폴더 감시와 OCR 검사 연결
-> 업로드 모니터링 대시보드 확장
-> 문서 처리 플랫폼 NewsFlow로 발전
```
